## 第二章 信息收集管理

### 2.1 收集原则

依据《个人信息保护法》第 6 条，童慧行严格遵循以下原则：

1. **合法正当**：有合法基础
2. **最小必要**：只收集服务必需的
3. **明确告知**：收集前明确告知
4. **知情同意**：取得用户单独同意
5. **儿童特别**：14 岁以下须监护人单独同意

### 2.2 收集的字段清单

完整字段清单详见附件 7（《童慧行隐私政策》）。

#### 2.3.2 收集合法性校验

**后端代码位置：** `apps/auth-service/src/utils/consent.js`

```javascript
// 14 周岁以下必须家长同意
function requiresParentConsent(userId) {
  // 1. 已 verified 的不需要
  if (consent && consent.verified === 1) return false;
  // 2-4. 用户自己生日
  const userAge = computeAge(user.birthday);
  if (userAge === null) return true;  // 缺失从严
  if (userAge < 14) return true;
  // 5. 关联儿童
  for (const child of children) {
    const age = computeAge(child.birthday);
    if (age === null || age < 14) return true;
  }
  return false;
}
```

**前端实施：** `apps/main-site/src/components/AuthModal.vue` + `App.vue`

- 三处登录流程统一按后端 `requiresParentConsent` 裁决
- App.vue 全局门卫：登录后自动检查，true 时强制弹 parent-consent
- 强制门卫模式：未完成家长同意，弹窗无法关闭

#### 2.3.3 收集的告知

## 第三章 隐私保护

### 3.1 全站 HTTPS

- 边缘层：阿里云 ECS 部署 Caddy 容器处理 TLS 终结
- 证书：阿里云 SSL 免费证书，自动续期
- 协议：TLS 1.3 + HTTP/2
- HSTS：已启用
- 明文 HTTP：301 跳转到 HTTPS

### 3.2 Cookie 安全

**代码位置：** `apps/auth-service/src/routes/auth.js`

```javascript
res.cookie('access_token', token, {
  httpOnly: true,        // 禁止 JS 读取
  secure: true,          // 仅 HTTPS
  sameSite: 'lax',       // 跨站请求保护
  domain: '.grandand.com',
  path: '/',
  maxAge: 7 * 24 * 60 * 60 * 1000
});
```

- ✅ HttpOnly：禁止 JavaScript 读取，防 XSS
- ✅ Secure：仅 HTTPS 传输
- ✅ SameSite=Lax：防 CSRF
- ✅ Domain 共享：跨子站单点登录

### 3.3 密码安全

**代码位置：** `apps/auth-service/src/routes/auth.js:14-22`

```javascript
const passwordHash = await bcrypt.hash(password, 10);
// 登录验证
bcrypt.compareSync(password, user.password_hash);
```

- 算法：bcrypt（10 轮 cost）
- **明文密码从不存储**
- 最小长度 6 字符

### 3.4 HTTP 安全头

**代码位置：** `apps/auth-service/src/index.js:14`

```javascript
app.use(helmet());
```

helmet 自动添加：
- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY
- X-XSS-Protection: 1; mode=block
- Strict-Transport-Security
- Referrer-Policy

### 3.5 CORS 配置

**代码位置：** `apps/auth-service/src/index.js:15-19`

```javascript
app.use(cors({
  origin: config.cors.origin,  // 白名单
  credentials: true,           // 允许 Cookie
}));
```

- Origin 白名单（仅 grandand.com 子域）
- 允许携带凭证

- 完整版：附件 7（《童慧行隐私政策》）
- C 端友好版：附件 8（《监护人须知》）

- 《儿童个人信息网络保护规定》（2019）：详细规定儿童信息保护要求

### 4.3 家长验证流程

```
1. 14 岁以下用户注册
   ↓

   ↓

   ↓

   ↓
5. 家长输入验证码
   ↓

   ↓

   ↓
8. 允许进入主界面
```

### 4.5 儿童信息保护专项措施

1. **不收集真实姓名、真实头像、真实手机号**
2. **社交功能默认关闭**（不提供陌生人私聊）
3. **位置信息不收集**
4. **不展示儿童真实信息**
5. **儿童内容发布全部必审**
6. **儿童数据访问双重审计**
7. **监护人可随时撤回同意**
8. **撤回后儿童档案立即删除**

## 第五章 敏感信息管控

### 5.1 敏感信息分类

| 分类 | 字段 | 保护等级 |
|---|---|---|
| **特别敏感** | 14 岁以下儿童身份信息、家庭关系 | 最高 |
| **高敏感** | 手机号、身份证号（未启用）| 高 |
| **中等敏感** | 头像、学习数据、行为记录 | 中 |
| **低敏感** | 公开昵称 | 低 |
| **公开** | 诗词/国学/通识内容 | 无 |

### 5.2 敏感信息存储

| 字段 | 加密 | 存储位置 |
|---|---|---|
| 密码 | bcrypt 10 轮 | SQLite |
| 手机号 | 明文（仅 1 个字段，未来加字段级加密）| SQLite |
| 头像 | HTTPS 传输，OSS 加密存储 | 阿里云 OSS |
| 学习数据 | 明文 | SQLite |
| 儿童数据 | 明文 + 业务层访问控制 | SQLite |

**未来改进方向**：
- 手机号字段级加密（AES-256）
- 身份证号字段级加密
- 备份文件加密

### 5.3 敏感信息访问控制

#### 5.3.1 权限分级

| 级别 | 可访问 | 审批人 |
|---|---|---|
| L4 最高 | 全部 | 总负责人 + 法人 |
| L3 高度 | 儿童数据 | 总负责人 |
| L2 敏感 | 个人信息 | 数据安全负责人 |
| L1 一般 | 业务数据 | 部门负责人 |
| L0 公开 | 公开数据 | 自动 |

#### 5.3.2 多因素认证

- L3 及以上访问须双因子认证
- 实施：JWT（已有）+ 短信验证码（开发中）

### 5.4 操作审计

**代码位置：** 各业务模块的 `db.prepare()` 调用

所有数据库操作通过 better-sqlite3 预编译语句执行，便于审计：

```javascript
// 全部走 prepare，不支持字符串拼接
db.prepare('SELECT * FROM users WHERE id = ?').get(userId)
```

- 防 SQL 注入
- 慢查询日志（计划接入）

### 5.5 备份与恢复

- **备份频率**：每日凌晨 3 点全量
- **备份位置**：阿里云 OSS（华北 2）
- **保留期限**：30 天滚动
- **恢复演练**：每半年 1 次

## 第六章 举报与投诉处理

### 6.1 受理渠道

| 渠道 | 用途 | 优先级 |
|---|---|---|
| 站内"举报"按钮 | 内容违规举报 | 最高 |

| 邮件 [privacy@grandand.com] | 个人信息保护 | 高 |
| API `POST /api/moderation/report` | 开发者举报 | 中 |

### 6.2 内容审核系统

**代码位置：** `apps/moderation/server/`

- **服务端口**：3020
- **审核算法**：DFA 敏感词检测（确定有限状态自动机）
- **敏感词库**：`apps/moderation/server/words.js`
  - 6 大类：粗口、暴力、成人、霸凌、垃圾广告、敏感话题
  - 200+ 敏感词
- **审核端点**：`POST /api/moderation/check`（服务间调用）
- **人工复审**：管理员后台，敏感内容抽检

### 6.3 举报处理流程

```
用户举报（站内/邮件/API）
  ↓
[后端] 写入 reports 表
  ↓
[后端] 触发内容审核
  ↓
审核通过 → 删除/屏蔽 + 处罚
审核未通过 → 记录 + 告一段落
  ↓
[管理员] 人工复核（24h 内）
  ↓
反馈举报人
```

**时限**：
- 涉儿童：4 小时内处理
- 其他：24 小时内处理
- 复杂投诉：15 个工作日

### 6.4 申诉机制

- 被处罚用户可申诉
- 端点：`POST /api/moderation/appeal`
- 时限：15 个工作日内重新审核
- 申诉成功：撤销处罚、恢复内容
- 申诉失败：保留处罚

### 6.5 涉儿童举报特别处理

详见附件 5（《儿童保护操作指南》）第七章"紧急情况"。

**任何涉及儿童被诱骗、涉儿童色情、教唆自残/自杀/离家出走的举报**：
- 立即删除内容
- 永久封禁相关账号
- 通知监护人
- 24 小时内报告网信部门
- 涉嫌犯罪报告公安机关

## 第七章 法律风险防控

### 7.1 主要法律风险

| 风险 | 等级 | 防控措施 |
|---|---|---|
| 儿童信息违规收集 | 极高 | 强制 parent-consent |
| 未成年人消费纠纷 | 低 | 平台无付费功能 |
| 知识产权侵权 | 中 | 公示投诉渠道 |
| 内容违规 | 中 | DFA 敏感词 + 人工复审 |
| 数据泄露 | 中 | 全站 HTTPS + 加密 + 备份 |
| 跨境传输违规 | 低 | 主要存境内，仅 Edge TTS 文本 |

## 第八章 应急响应

### 8.1 应急预案

完整预案见附件 3（《数据安全事件应急预案》）。

### 8.2 事件分级

| 级别 | 涉及用户量 | 涉及儿童 | 处置 |
|---|---|---|---|
| I 级 | ≥ 100 万 | 涉及 | 1 小时内报告网信办 |
| II 级 | 10 万-100 万 | 涉及 | 2 小时内报告 |
| III 级 | 1 万-10 万 | 不涉及 | 24 小时内报告 |
| IV 级 | < 1 万 | 不涉及 | 内部记录 |

### 8.3 报告义务

依据《个人信息保护法》第 57 条：
- **数据泄露**：立即采取补救措施，通知个人
- **涉儿童**：无论级别，1 小时内向网信部门报告
- **可能危害**：72 小时内通知用户

### 8.4 应急组织

- 总指挥 + 技术 + 安全 + 客服 + 法务 + 公关
- 7×24 小时响应
- 详见附件 3

### 8.5 应急演练

- 桌面演练：每季度
- 实战演练：每年
- 最近演练：2026 年 6 月
