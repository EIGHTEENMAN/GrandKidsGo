# 潮玩换智能管理后台设计方案

## 核心理念
**CEO只审不操**——所有分析、方案、执行由AI总监完成，CEO只做审核与决策。
**CEO不盯后台**——所有需要CEO处理的事项通过邮件/短信/微信主动推送，看到信息再到工作台处理。

## 三源数据铁律（硬性规则）
任何方案/报告必须同时包含三个维度的数据，缺少任一项自动驳回。每个数据点必须标注来源格式。

## 8位总监架构
| # | 总监 | 板块 | 职责 |
|---|------|------|------|
| 1 | TD | 平台监控 | 服务器/数据库/API/安全/性能/灾备 |
| 2 | OD | 智能运营 | 交换业务管理 + 店铺业务管理 + 产品供应链（A虚拟/B一件代发/C自营） |
| 3 | MD | 智能营销 | 定目标→出方案→执行→复盘→ROI报告 |
| 4 | FD | 智能财务 | 收入/成本/利润/报表/经营分析 + 预算池管理 |
| 5 | CD | 内容运营 | 话题/UGC/KOL/内容审核/频道推荐 |
| 6 | GD | 用户增长 | 引流策略(5大渠道15子渠道) + **用户行为分析**(页面热力/转化漏斗/路径) + 获客分析+裂变+分层+实验+召回 |
| 7 | CRD | 风控合规 | 商品审核/交易风控/规则引擎/数据合规 + 平台资质合规(小程序/ICP/商标/新规跟踪) |
| 8 | CS | AI客服 | 智能客服模型+人工介入闭环 |

## 目标管理体系
CEO出愿景 → MD+FD联合制定目标树 → CEO审批 → 各总监对齐执行 → 月度MD+FD复盘

## 通知触达机制
- 🔴紧急（服务器宕机/安全事件）：短信+邮件+微信三重轰炸，AI先止损
- 🟡待审批（需决策）：邮件+微信，标题标「审批」，点链接30秒处理完
- 🟢阅示报告（只需知道）：邮件摘要300字，瞄一眼即可
- 勿扰时段23:00-07:00仅紧急通知
- 同类通知30分钟内自动合并

## 用户行为分析（GD主责，全总监共享）
自动产出：页面热力Top5 / 转化漏斗 / 流失点定位 / 路径分析 / 搜索分析
各总监在各自报告中直接引用GD分析数据，交叉参照形成方案依据

## 预算与支出管控体系
FD统一管预算池，池内自动审批阈值，超支预警，事后报告

## 数据采集体系
外部：爬虫中心监控1688/闲鱼/泡泡玛特/得物/小红书/微博/抖音/微信指数
内部：MySQL直连 + 埋点系统 + ELK日志

## 实施路线
Phase 1基建期 → Phase 2总监上线期 → Phase 3扩展期 → Phase 4智能进化期

**Why:** CEO不能实时盯后台，需要主动通知触达。用户行为分析是转化优化的核心数据基础，GD作为增长总监最适合承担。

**How to apply:** 搭建CEO工作台时，通知模块是第一优先级——邮件推送+审批链接是核心交互，不是工作台页面本身。
