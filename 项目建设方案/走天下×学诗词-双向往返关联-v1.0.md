# 走天下 × 学诗词 双向往返关联方案 v1.0

> 项目：apps/travel-guide（走天下）+ apps/xueshici（学诗词）
> 背景：用户在走天下看到李白登黄鹤楼的地方（武汉黄鹤楼），可以一键看李白写的《黄鹤楼送孟浩然之广陵》全文；反过来在学诗词读到"姑苏城外寒山寺"，可以跳转到走天下苏州寒山寺的地点详情 + 周边亲子攻略
> 关键约束：xueshici 是纯前端 SPA（无 DB/API），travel-guide 有 DB/API；两者不同域（travel.grandand.com / xueshici.grandand.com），通过 hash 路由跳转

## Context

### 学诗词现状（调研结论）
- 纯前端 Vue 3 + Vite SPA，**无数据库、无 Prisma、无 API 后端**
- 域名 `xueshici.grandand.com`，已部署线上（Caddy SSL + nginx 静态文件）
- 详情页 URL：`https://xueshici.grandand.com/#reader/{poemId}-1`
- **诗词数据无地点字段**：`Poem` 类型只有 `id/title/author/dynasty/category/summary/tags/sections`，无 `location/city/geo`
- 2026+ 首诗词，按 21 个 chunk 懒加载（`poem-chunks/chunk-0.ts` ~ `chunk-20.ts`）
- 有 `/api/search?q=` 全站共享搜索接口（走 auth.grandand.com，非学诗词专属）

### 走天下现状（可复用）
- Next.js 14 + Prisma + PostgreSQL
- `Spot` 表：lat/lng/cityId/name/tags/kidHighlights/kidScore（详细地点）
- `City` 表：51 城 + name/lat/lng
- `Guide` 表：城市级攻略
- `/place/[type]/[id]` 详情页（蓝青，已设计 v1.0 方案）
- `/guides/[id]` 详情页（v1.0 方案设计中）

### 核心难点
1. **学诗词无地点字段** → 需要建立"诗词 ↔ 地点"映射表（纯右侧实现，不动学诗词代码）
2. **跨域跳转** → 走天下与学诗词不同域，通过 `window.open` / `<a target="_blank">` 跳转
3. **双向往返** → 走天下能跳学诗词，学诗词也能跳走天下（学诗词侧需要加静态数据链接配置）

## 1. 数据库 Schema 改动

### 1.1 新增 `PoemLocation` 映射表（走天下侧，唯一的数据底座）

```prisma
enum PoemLinkType {
  EXACT_CITY       // 精确城市关联（如"姑苏"→苏州）
  LANDMARK         // 地标/景点（如"黄鹤楼"→武汉黄鹤楼）
  AUTHOR_HOMETOWN  // 作者籍贯（如"王安石→抚州"→城市）
  AUTHOR_POSTING   // 作者任职/游历地（如"苏轼→杭州"）
  SCENE            // 场景地点（如"浔阳江头"→九江）
}

model PoemLocation {
  id          String      @id @default(uuid())
  poemId      Int                  // 学诗词 poem.id（1-based，2026+）
  poemTitle   String      @map("poem_title")              // 诗词标题（反范式，冗余）
  poemAuthor  String      @map("poem_author")             // 作者名（反范式）
  linkType    PoemLinkType @map("link_type")
  placeType   String      @map("place_type")              // 'city' | 'sight'
  placeId     String      @map("place_id")                // City.id 或 Spot.id
  placeName   String      @map("place_name")              // 反范式，冗余
  cityId      String?     @map("city_id")                 // 城市 ID
  verseLine   String?     @map("verse_line")              // 原诗句，如"姑苏城外寒山寺"
  confidence  Int         @default(3)                     // 1-5 关联置信度（1=A I 提取 3=人工确认 5=原文明确）
  source      String      @default("ai_draft")            // ai_draft | manual_verified | user_contributed
  createdAt   DateTime    @default(now()) @map("created_at")

  @@unique([poemId, linkType, placeType, placeId])        // 防止重复
  @@index([poemId])
  @@index([placeType, placeId])
  @@index([cityId])
  @@map("poem_locations")
}
```

**数据来源**（按优先级）：
1. **人工标注**（P0）：精选 50-100 首有明确地标的古诗（黄鹤楼/岳阳楼/庐山/寒山寺/滕王阁等），手写映射
2. **AI 提取**（P1）：从 `poems-meta.ts` + `poem-chunks/chunk-*.ts` 扫描诗句中的地名，用 AI 识别"西湖/黄鹤楼/浔阳"等地标 → 匹配走天下 Spot/City
3. **用户贡献**（P2）：走天下详情页底部"关联古诗词"按钮 + 表单提交

### 1.2 走天下 `Spot` 表加诗词关联字段（可选，非必需）

注：不直接改 `Spot` 模型。所有关联通过 `PoemLocation` 表查询（`: placeType='sight' AND placeId=spot.id`）。

### 2.2 匹配走天下 Spot/City

映逻辑：
- `placeType='sight'` → `Spot` 表 `WHERE name LIKE '%寒山寺%' OR tags @> '{黄鹤楼}'`
- `placeType='city'` → `City` 表 `WHERE name='苏州'`（走天下 51 城已写入）
- 匹配不上时走 `cityId` 城市级关联（降级方案）

### 3.2 走天下详情页消费接口

`/place/[type]/[id]/route.ts` 升级（P0 已有计划，加一个 `poems` 字段）：
```ts
const poems = await prisma.poemLocation.findMany({
  where: {
    OR: [
      { placeType: type, placeId: id },
      { cityId: place.cityId },
    ],
  },
  take: 5,
  orderBy: { confidence: 'desc' },
});
```

返回数据中的 `poems` 数组字段：
```ts
poems: [{ poemId, poemTitle, poemAuthor, linkType, verseLine, url, confidence }],
```

### 3.3 学诗词侧无需 API 改动（纯静态 SPA 无需后端）

学诗词侧通过**配置文件 + 静态 JSON**接收走天下数据：
- 在 `apps/xueshici/src/data/poem-locations.json` 存放走天下场景的同步数据
- 学诗词阅读器（`Reader.vue`）读取 `window.__POEM_LOCATIONS__`（由数据同步脚本注入 build）
- 或者学诗词在页面读取 `travel.grandand.com/api/poems/by-id?id={poemId}`（跨域 JSONP）

**推荐方案**：走天下侧 `/api/poems/by-id` 返回该诗的关联地点列表，学诗词 reader 通过 `fetch` 跨域调用（CORS 开放 `xueshici.grandand.com` 即可）。

## 7. 风险 & 取舍

- **跨域 CORS**：走天下 `next.config.mjs` 加 `Access-Control-Allow-Origin: https://xueshici.grandand.com`；或者学诗词 reader 用 `<a>` 直跳（不走 fetch）避免 CORS
- **学诗词是 SPA**：没有 vue-router，hash 路由外部可直接跳转 `xueshici.grandand.com/#reader/{id}-1`— 兼容性已验证
- **诗词无数据接口**：走天下侧写死 `poemId→title/author`（反范式冗余在 `PoemLocation`），不走学诗词 API（因不存在）
- **关联置信度**：AI 提取可能误匹配（"洛阳"的"洛"字与别的词重叠），设 `confidence` 字段，UI 低于 3 不显示
- **走天下 CORS 安全**：只开放 `xueshici.grandand.com` 一个域，不开放泛域名
- **双向往返闭环**：走天下跳学诗词（新标签页），学诗词跳走天下（新标签页），不走内嵌 iframe
