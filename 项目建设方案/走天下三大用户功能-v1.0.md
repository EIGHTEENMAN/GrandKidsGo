# 走天下三大用户功能 v1.0 落地方案（足迹地图 + 儿童画廊 + 孩子说）

> 项目：apps/travel-guide（Next.js 14 + Prisma + PostgreSQL）
> 触发：常见问题页 `src/app/faq/page.tsx:12-14` 承诺了"足迹地图/儿童画廊/孩子说"3 个用户功能，但代码库 0 实现
> 调研：2026-07-22 已完成现状摸底（DB 38 表/API 13 类/页面 17 个全部无对应）
> 目标：补齐 3 大功能的产品债务，与现有 v4.0 UI/合规/勋章体系深度复用

## Context

`apps/travel-guide/src/app/faq/page.tsx:12-14` 把"足迹地图/儿童画廊/孩子说"作为用户功能宣传文案：

```
"足迹地图是让孩子记录去过的地方的功能，每到一个新地方可以打卡签到，收集属于自己的旅行地图。"
"儿童画廊是专门为孩子设计的照片展示区，孩子可以上传旅行中的照片并配上说明。"
""孩子说"记录孩子在旅行中的童言趣语，保存这些珍贵的成长瞬间。"
```

但全代码库 grep 命中只有这 3 行宣传文案 + 1 个"首都足迹"勋章名（不是功能）+ 几处营销 slogan。**0 实现**。

好消息：DB 38 张表里有 `TravelRecord`（行程级足迹雏形）+ `PlanMedia`（结构够用的照片表）+ `ConsentRecord`/`RetractionLog`（隐私硬合规）+ `ChildProfile` + `TravelActivity` + `/api/upload/sts`（OSS 凭证）—— 这些都是 3 大功能的现成积木。

## 1. 调研结论（每个功能现状 + 难度 + 复用度）

### 1.1 足迹地图

**现状**
- DB：无 `Footprint`/`Checkin` 模型；`TravelRecord` 表存在但代码无引用（只是 scoring 注释）
- API：无 `/api/footprints*`；最接近的是 `/api/user/travel-records`（基于已完成 planRecord 推导城市级）
- 前端：无 `/footprints` 路由；首页无地图板块
- 仅有"足迹"语义在勋章名（"首都足迹"/"江南水乡"/"西北豪情"），是后置统计不是用户主动打卡

**缺失**
- 地点级模型（含 cityId/spotId/lat/lng/checkinAt/childId）
- 地图组件（Leaflet / 高德 JS API / Mapbox — `.env:16` AMAP_API_KEY 仍空）
- 打卡交互（每到新地方一次签到）

**复用度**：`City`/`Spot` 经纬度、`ChildProfile` 体系、`ConsentRecord` 隐私框架、勋章 criteria 改造

**难度评分：3/5**（地图组件 + 重复打卡幂等性 + 隐私门槛）

### 1.2 儿童画廊

**现状**
- DB：`PlanMedia` **结构上够用**（ossKey/ossUrl/caption/spotId/visibilityLevel/caption）
- API：仅 `/api/upload/sts` 签 mock OSS 凭证；**无任何 GET/POST PlanMedia 的 API**
- 前端：无 `/gallery` 路由；计划详情页是 stub
- `visibilityLevel` 字段已有（`private`/`public`/`community`），合规基础在

**缺失**
- user-facing 的 PlanMedia 列表 / 上传 / 删除 API
- "孩子视角"语义（需 `childId` 字段）
- 画廊 UI（瀑布流 + 大图预览）
- 上传组件（OSS 直传 + 进度条）

**复用度最高**：`PlanMedia` 表结构、`/api/upload/sts`、`ConsentRecord` + `RetractionLog`、`visibilityLevel` 字段、TipTap 富文本

**难度评分：2/5**（schema 改动最小；隐私/上传链路已就绪）

### 1.3 孩子说

**现状**
- DB：**完全无模型**（无 `ChildSaying`/`ChildQuote`/`KidSay`/`Memorial`）
- API：无 `/api/child-sayings` `/api/quotes`
- 前端：无 `/child-sayings` 路由；首页 "孩子说好" 标题是营销文案
- 沾边的 `ChildRating` 是结构化评分（physicalState/emotionalPeak），不是童言趣语

**缺失**
- 整套表 / API / UI
- 隐含需求：可文字 + 可语音（v1.4 第十五节 B 已规划 `ai_voice_analysis` 同意项）

**复用度**：`ChildProfile`、`ConsentRecord`/`RetractionLog` 隐私硬合规、`TravelActivity` 社区动态、勋章体系

**难度评分：3.5/5**（schema/API 简单；语音能力是全新工作流；隐私门槛最高，因为涉及"孩子言论"）

## 3. 公共基础设施（3 功能共用）

### 3.1 隐私硬合规（v1.4 第十五节 B 钉死）

所有 3 个功能都涉及孩子数据，**必须**走 `ConsentRecord` 框架：
- `scope` 字段：`"publish_to_public"` / `"publish_to_community"` / `"voice_analysis"`
- 发布到公开/社区必须先有 `ConsentRecord.status='granted'`
- 撤回走 `/api/consent/revoke` 的 `RetractionLog` 模式（软删 + 审计日志）

实现模板：`apps/travel-guide/src/app/api/consent/revoke/route.ts`

### 3.2 OSS 上传链路（已有）

`/api/upload/sts` 已签 mock 凭证（`AMAP_API_KEY=...` 注释里留接入 TODO）：
- 复用 STS 凭证 + 前端 OSS 直传
- 新增 `ChildSaying.voiceOssKey` / `PlanMedia.ossKey` 字段

### 3.3 勋章体系联动（已有）

`apps/travel-guide/src/lib/badge-defs.ts` 已有"首都足迹"等勋章定义：
- 把勋章 criteria 改为读 `Footprint` / `ChildSaying` / `PlanMedia` 表
- 新增勋章："金句收藏家"/"小小摄影师"/"小小旅行家"

### 3.4 孩子身份绑定（已有）

`ChildProfile` 表 + `ChildExpectation` + `ChildFeelingProfile` + `ChildRating` 已成体系
- 三个功能都必须挂 `childId`（强制家长选孩子后才能使用）

## 4. 实施步骤（按推荐优先级）

### P0 — 儿童画廊（先发，最高复用）

**目标**：补齐 `PlanMedia` 的 CRUD 接口 + 新增 `/gallery` 页面，与现有 `/plan/[id]` 的"移动端继续编辑"对齐到 PC 端可见

1. **Schema 调整**：
   ```prisma
   model PlanMedia {
     // ... 既有字段
     childId       String?  @map("child_id")  // 新增：孩子绑定
     sourceType    String   @default("plan")  // 新增：plan | gallery
     visibilityLevel String @default("private") @map("visibility_level")
   }
   @@index([childId, visibilityLevel, createdAt(sort: Desc)])
   ```

2. **新增 API**：
   - `GET /api/gallery?childId=&visibilityLevel=`（列表，按 visibilityLevel 过滤）
   - `POST /api/gallery`（上传完成后回写 PlanMedia）
   - `DELETE /api/gallery/[id]`（撤回 → RetractionLog）
   - `POST /api/gallery/[id]/visibility`（改 private/community/public）

3. **新增前端页面** `apps/travel-guide/src/app/gallery/page.tsx`：
   - 瀑布流（3 列）
   - childId 切换器（顶部）
   - visibility 切换（私人/社区/公开）
   - 上传弹窗（OSS 直传，复用 `/api/upload/sts`）
   - 大图预览 + caption 编辑
   - 撤回按钮

4. **隐私前置**：发布到 community/public 前强制 `ConsentRecord` 检查

### P1 — 足迹地图

**目标**：补地点级打卡 + 地图可视化（PC 端用高德地图组件，移动端暂用列表视图）

1. **Schema 新增**：
   ```prisma
   model Footprint {
     id        String   @id @default(uuid())
     userId    String   @map("user_id")
     childId   String   @map("child_id")  // 强制
     cityId    String?  @map("city_id")
     spotId    String?  @map("spot_id")
     lat       Float?
     lng       Float?
     checkinAt DateTime @map("checkin_at")
     mood      String?  // emoji-free 文字标签
     note      String?
     visibilityLevel String @default("private") @map("visibility_level")
     createdAt DateTime @default(now())
     @@unique([userId, childId, spotId, checkinAt])  // 防止重复打卡
     @@index([userId, childId, createdAt(sort: Desc)])
     @@map("footprints")
   }
   ```

2. **新增 API**：
   - `GET /api/footprints?childId=&cityId=`（列表）
   - `POST /api/footprints`（打卡签到）
   - `DELETE /api/footprints/[id]`（撤回）
   - `GET /api/footprints/stats?childId=`（城市分布 + 累计勋章进度）

3. **新增前端页面** `apps/travel-guide/src/app/footprints/page.tsx`：
   - **地图视图**（PC）：高德 JS API（需 AMAP_API_KEY） + city marker 聚合 + 时间线
   - **列表视图**（移动 fallback）：按城市分组的卡片列表
   - childId 切换器
   - 打卡按钮：当前位置 → 选城市/景点 → 写 note + mood
   - 与现有勋章"首都足迹"criteria 联动

4. **关键决策**：AMAP_API_KEY 还没配（`.env:16` 空）。要么配真实 key，要么先用静态地图截图 fallback（无交互但能展示分布）

### P2 — 孩子说

**目标**：纯文字版先上线，语音能力后续迭代

1. **Schema 新增**：
   ```prisma
   model ChildSaying {
     id               String   @id @default(uuid())
     userId           String   @map("user_id")
     childId          String   @map("child_id")
     planRecordId     String?  @map("plan_record_id")
     spotId           String?  @map("spot_id")
     text             String                          // 童言趣语原文
     voiceOssKey      String?  @map("voice_oss_key")
     voiceDurationSec Int?     @map("voice_duration_sec")
     mood             String?                          // "开心"/"困惑"/"惊讶"
     shareScope       String   @default("private") @map("share_scope")
     createdAt        DateTime @default(now())
     @@index([childId, createdAt(sort: Desc)])
     @@map("child_sayings")
   }
   ```

2. **新增 API**：
   - `GET /api/child-sayings?childId=&shareScope=`（列表）
   - `POST /api/child-sayings`（提交）
   - `DELETE /api/child-sayings/[id]`
   - `POST /api/child-sayings/[id]/share`（改 shareScope）
   - `GET /api/child-sayings/random`（抽一句做激励语录）

3. **新增前端页面** `apps/travel-guide/src/app/child-sayings/page.tsx`：
   - 时间线列表（按时间倒序）
   - 文字输入 + 表情选择 + 场景标签
   - 详情页 `/child-sayings/[id]` 卡片视图（适合分享到小红书）
   - 分享范围：私人 / 社区 / 公开
   - 随机金句 banner（用 `/api/child-sayings/random`）

4. **勋章联动**：新增"金句收藏家"勋章（N 条收录）
