# 走天下攻略详情页 v1.0 全栈改造方案

> 项目：apps/travel-guide（Next.js 14 + Prisma + PostgreSQL），部署域名 `travel.grandand.com`
> 路由：`/guides/[id]`（注意是复数），API：`/api/guides/[id]`
> 目标：v1 老 UI（绿色 prose-green + 旧数据契约 + 404 bug）→ v4.0 蓝青渐变 + 9 板块 UI + 大图轮播 + 双维度评分 + 评论 + 关联攻略 + 作者主页 + fork CTA
> 关键约束：UI 调性（`from-blue-600 via-cyan-600 to-teal-600`）、零 emoji、HTML 白名单渲染（手写 parser，零依赖）

## Context

攻略详情页 `/guides/[id]` 当前是 v1 老 UI（绿色 `prose-green` + 旧数据契约），且**所有攻略卡片点进去 404**——因为列表/首页/Wizard 用了单数 `/guide/${id}`，实际是复数 `/guides/[id]`。同时 `/api/guides/[id]` 已切到 v2 数据契约（`contentHtml: string` + `stats: {view, save, like}` + `sourcePlan.timelineBlocks`），但前端详情页还在读旧 `sections/ratings/comments`（POST 接口 404）。本次按 v4.0 蓝青渐变重做，加双维度评分 + 评论 + 关联攻略 + 作者主页 + fork CTA，修单数/复数路径 bug 与 schema 漂移（`cover_images` nullable、`GuideFork` 表没建、`GuideDraft` 零引用）。

业务目标：让妈妈在攻略详情页能
1. 看完整攻略（大图轮播 + 正文 + 行程 Day-by-Day + 作者信息）
2. 一键 fork 成自己的计划（CTA 按钮）
3. 点赞 / 收藏（like/save toggle）
4. 双维度打分（大人 + 孩子，复用 PlaceReview 模式）
5. 评论互动
6. 跳到作者主页看他其他攻略
7. 分享攻略

实施范围：**全栈改造**（schema + API + UI），分 P0-P4 渐进上线。

## ⚠️ 严重预警：当前 Prisma 三方漂移

调研发现严重漂移：
- `prisma/schema.prisma` = 7/21 版（含 PlaceReview/PlaceViewLog/GuideFork 等 v3 模型）
- `node_modules/.prisma/client/schema.prisma` = 7/18 旧版（仅 812 行，无 v3 delegate）
- DB 仅应用 7/18 五个 migration
- **后果**：`/api/guides/fork` 调 `prisma.guideFork.create` 必然崩（旧客户端根本没这 delegate）
- **波及**：任何 v3 模型相关 API 都不可靠（GuideFork / PlaceViewLog / PlaceReview 等）

**P0 第一步必须先修复三方漂移**，否则后续 Prisma 操作不可预期。详细修复步骤见 §6 P0 步骤 1。

## 1. 数据库 Schema 改动（`prisma/schema.prisma`）

### 1.1 修 schema 漂移（P0 + P4）

```prisma
// 当前 schema 看似 required，但 SQL 是 nullable（4 行 0 行有图）
// P4: 加 NOT NULL + 默认值 '{}' + DB 回填
model Guide {
  // ...
  coverImages String[] @default([]) @map("cover_images")  // 现在实际是 nullable，DB 回填
}

// P0 必须：GuideFork 表在 schema 里有但 DB/Prisma client 都没建
// → 立即加 migration 建表 + 索引（否则 fork CTA 调用即崩）
model GuideFork {
  id             String   @id @default(uuid())
  sourceGuideId  String   @map("source_guide_id")
  forkedByUserId String   @map("forked_by_user_id")
  planRecordId   String?  @map("plan_record_id")
  modified       Boolean  @default(false)
  createdAt      DateTime @default(now()) @map("created_at")

  @@index([sourceGuideId])
  @@index([forkedByUserId])
  @@map("guide_forks")
}
```

### 1.2 新增 `GuideReview` 双维度评分（P1，复用 PlaceReview 模式）

```prisma
model GuideReview {
  id             String   @id @default(uuid())
  guideId        String   @map("guide_id")
  userId         String   @map("user_id")
  adultRating    Int      @map("adult_rating")      // 1-5 大人评分（实际体验）
  childRating    Int?     @map("child_rating")      // 1-5 孩子评分（可空）
  childAgeMonths Int?     @map("child_age_months") // 评分时孩子月龄
  text           String?
  tags           String[]                              // "行程紧凑" / "孩子不适合" 等
  visitDate      DateTime? @map("visit_date")
  status         String   @default("published")
  createdAt      DateTime @default(now()) @map("created_at")

  @@index([guideId, createdAt(sort: Desc)])
  @@index([status])
  @@map("guide_reviews")
}
```

### 1.3 新增 `GuideComment` 评论（P1）

```prisma
model GuideComment {
  id        String   @id @default(uuid())
  guideId   String   @map("guide_id")
  userId    String   @map("user_id")
  content   String                              // 纯文本，前端不渲染 HTML
  status    String   @default("published")    // published | hidden
  createdAt DateTime @default(now()) @map("created_at")

  @@index([guideId, createdAt(sort: Desc)])
  @@map("guide_comments")
}
```

### 1.4 不动 `GuideDraft`

`GuideDraft` 表 schema 已有但零运行时引用。**P3 再决策**（评估是否要落地 AI 攻略向导流程）。本方案暂不涉及。

## 2. 数据安全：contentHtml 白名单清洗

### 2.1 后端清洗策略（手写 parser，零依赖）

新建 `src/lib/sanitize.ts`：

```ts
// 白名单：h2/h3/p/ul/li/ol/strong/em/img/a/br/blockquote/hr
// 属性白名单：img[src,alt]、a[href,rel,target]
// 强制：a 标签加 target=_blank rel=noopener noreferrer
// 删除：script/style/iframe、所有 on* 事件、javascript: URL、style 属性、data-* 属性
// 长度上限：50000 字符
export function sanitizeHtml(input: string): string { ... }
```

**不引 DOMPurify**（手写，零依赖，已在生产环境稳定使用）。

### 2.2 评论/评分文本防护
- 评论纯文本存储（无 HTML 渲染）
- 评分 text 字段：纯文本，前端 `\n` 换行
- 长度限制：comment ≤ 500 字符，review text ≤ 1000 字符
- tags 过滤：白名单攻略专属词库（`/src/lib/guide-tags.ts`）

### 2.3 防刷
- 评论：同一用户对同一 guide **1 天 1 条**
- 评分：同一用户对同一 guide **永久 1 条**（可改但少见）
- 发布：同一用户 **1 小时 2 篇**
- 实现：API 端查 `count where userId=X AND guideId=Y AND createdAt >= cutoff`

## 3. 后端 API 改动

### 3.1 升级 `GET /api/guides/[id]/route.ts`

**现状**：返回 `{id, title, coverImages, contentHtml, city, days, childAges, travelStyle, tags, publishedAt, stats:{view,save,like}, author:{id,nickname:'童慧行用户',avatar:null}, sourcePlan}`。

**升级后**：
```ts
{
  code: 'OK',
  data: {
    id, title, coverImages, contentHtml: sanitizeHtml(contentHtml),  // 后端清洗
    city: { id, name, kidHook, momHook, dadHook, coverImage, kidScore, momScore, dadScore },
    days, childAges, travelStyle, season, tags, publishedAt, createdAt,
    stats: {
      view, save, like,
      avgAdultRating: number|null,
      avgChildRating: number|null,
      ratingCount: number,
      commentCount: number,
    },
    author: { id, nickname, avatar, guideCount, totalLikes },  // 真实信息（接 user-service）
    isLiked: boolean,    // 基于当前用户（从 x-debug-user-id 取）
    isSaved: boolean,
    related: {            // P2 才有，P0 先空数组
      sameAuthor: [],
      sameCity: [],
    },
  }
}
```

实现要点：
- `sanitizeHtml()` 调用
- `Promise.all` 并发查：reviews 统计 + 当前用户 like/save + 相关攻略
- 作者信息：调 `src/lib/user-service.ts:41-69`（SQLite `users` 表）

## 7. 风险 & 取舍

- **关联完整性附注**（P2 作者主页时要清理）：travel-guide schema 无 User FK，`Guide.userId` 是 String（跨服务引用 auth-service SQLite `users`）。当前 DB 4 行 guide 中：
  - 1 行 userId = `169060b8-...`（在 auth.users 表真实存在）
  - 3 行 userId 是测试占位（`smoke-user` ×2 / `wave-d-user` ×1）
  - **所以 P2 实施作者主页时必须 fallback**：找不到真实用户的攻略显示"匿名妈妈"占位，否则前端会显示断裂的"undefined"昵称。建议 P2 加：mock 数据（如 mvp_id 列表）+ 真实查询双轨。
- **Prisma 三方漂移**：必须 P0 第 1 步修复，否则任何 Prisma 操作不可靠
- **GuideFork 必须 P0 修**：否则 fork 路径会直接 500
- **Guide 数据创建唯一入口**：`POST /api/guides/from-plan/[planId]/route.ts:109`，写 `body.coverImages ?? []`。**`prisma/seed.ts` + `04-import-db.ts` 不操作 Guide**（不要误以为有 seed 流），`snapshots/guides.json` 是导出快照、不是 seed 输入。所以 P3 实施发布攻略时**只需补 `POST /api/guides`**，无需补 seed。
- **cover_images 长度无 schema/code 强约束**：文档曾提"OSS key 数组、最多 3 张"，但代码未强制。P4 若加 `String[].length <= 3` 校验，需同时改 schema + API 校验 + DB 触发器；建议 P4 跳过（前端 ≤3 张即可，DB 不强约束）。
- **旧数据契约** `sections/ratings/comments`：完全忽略，dead code 删掉
- **DB 只有 4 行 guide**：详情页 mock 兜底 12 篇（已有）+ 评分/评论数走随机数演示
- **作者 userId 跨服务**：复用 `src/lib/user-service.ts` SQLite 直连（feed 已用，不引 HTTP）
- **手写 sanitize vs DOMPurify**：手写，零依赖（项目其他地方已用同模式）
- **路径 redirects + 源代码双修**：避免 SEO 损失
- **like/save count 标量 vs 实时 count**：当前是标量 + 触发器更新（避免每次 count(*)），保持现状
- **token 统一**：把 `localStorage.getItem('haodaer_token')` 切到 `sessionStorage.getItem('grandkidsgo_token')`（已有 helper）
