## 三、关键判断

### A. 标配（≥4 平台）→ **P0 必做**
1. **我的发布列表** — 5/5 平台都有"我的内容"列表（笔记/游记/点评）。童慧行只有 4 项数字、无列表页。优先做。
2. **消息/通知中心** — 5/5 都有。评论回复/点赞/收藏通知才有社交留存。
3. **设置中心**（资料/隐私/账号/通知） — 5/5 都有。后端 `TravelPrivacySetting` 表已存在，UI 全空白。

### B. 童慧行特色（亲子定位）→ **P0 必做**
4. **孩子档案管理** — 5 大平台都没有，是童慧行护城河。`ChildProfile` 表已存在，必须做多孩档案 + `ChildFeelingProfile` 摘要卡片。
5. **孩子说入口增强** — 已是核心功能，但当前只是数字卡片。应升级为"按孩子分组的童言趣语墙"，绑定孩子头像 + 月龄 + 景点。
6. **孩子感受画像** — `ChildFeelingProfile` 表已存在，是亲子独有。要展示活跃时段、热衷景点、哭闹触发器（合规脱敏后的统计视图）。
7. **勋章墙** — 5 平台只有 Tripadvisor Travelers' Choice，但童慧行的 `TravelBadgeDef` 表已就绪。直接复用，零新增表。

### C. 商业化导向 → **直接砍**
- 订单 / 钱包 / 优惠券 / 礼品卡 / 会员中心 / 拿去花 / 信用贷 — 携程、小红书全套，童慧行非商业化定位全部不学。
- 积分商城 — 穷游独有。童慧行有 `TravelBadgeExchange` 表但属勋章兑换体系，应该并入"勋章"模块而非独立展示。

### D. 数据驱动 → **P1 等数据沉淀**
8. **数据看板**（攻略 / 孩子 / 行程 三维度统计）— 复用 `PlaceViewLog` + `Guide` 已有表，新增 `profile/dashboard` 聚合接口即可上线。
9. **关注/粉丝** — `UserFollow` 表已有，可做但不优先。亲子 KOL 链尚未成型。
10. **行程模板我的收藏** — `ItineraryTemplate` 已有，"我的行程列表"打通直接可用。
11. **作品数据**（点赞/收藏/浏览） — `Guide` 表已存 `viewCount/saveCount/likeCount`，做发布者后台统计即可。

### E. 长尾 → **P2 长期规划**
12. 我的问答 — 平台依赖重（问答要有社区规模）
13. 创作中心 / 数据分析 — 等 KOL 群体成型再做
14. 实名认证 — 等有商家内容或敏感操作时再做

## 五、二次调研深挖：5 条产品差异化启示（2026-07-24 补充）

> 资料来源：WebSearch 二手 + 各家 curl 抓头部/页脚的直接验证。各家反爬墙（xhs SPA / ctrip /member 404 / tripadvisor captcha / qyer /member 下线）拦截了直接抓 HTML，所以以下结论以 WebSearch 业内公认结构 + 头部导航验证为准。

### 启示 1：孩子档案必须做结构化数据资产（学携程「常用出行人」）

**携程最强点**：常用出行人可存「宝宝」身份证/护照/年龄。下单时一键带入。
**小红书/马蜂窝/Tripadvisor/穷游都没做**：分散在笔记/评论里，无结构化字段。

→ **童慧行走天下要做的事**：
- 把 `ChildProfile` 表升级为「家庭档案中枢」：姓名/性别/出生年月/兴趣标签/护照号末 4 位/常用目的地偏好
- 不只是「个人中心展示」，**要在行程规划/做攻略/收藏时一键带入**
- 这是行前-行中-行后的全场景粘性金矿，比「孩子说」更高频调用

### 启示 2：足迹地图分三态（去过 / 想去 / 孩子想去）

**马蜂窝 + Tripadvisor 都有 Travel Maps**：用户点亮国家地图，热力图可视化，是个人主页最被截图分享的模块。

→ **童慧行走天下的差异化三层状态**：
| 状态 | 来源 | 触发 |
|------|------|------|
| **去过** | 攻略发布自动点亮 / Footprint 表 | 孩子说 + 儿童画廊 + 发布攻略 |
| **想去** | 家长收藏地点 | 收藏 + 攻略 PlanRecord |
| **孩子想去** | 儿童投票/想去心愿 | 童言趣语投票 |

第三层「孩子想去」是 5 平台都没有的独家锚点——亲子定位的天然护城河。

### 启示 3：贡献等级 + 勋章墙，不用稿费（学 Tripadvisor 非现金激励）

**Tripadvisor 0-6 级贡献者 + 6 类徽章**（Top Contributor/Reviewer/Photographer/Explorer）是「非现金激励」标杆。
**马蜂窝稿费/小红书蒲公英商单**有变现但跑偏做广告平台。
**穷游精华推荐无稿费**，但创作者上升通道窄。

→ **童慧行走天下定位**：
- **不学稿费/商单**（避免跑偏，妈妈爸爸写攻略不是为赚钱，是为记录孩子成长）
- **学 Tripadvisor 勋章 + 等级**：勋章名亲子化——「守护者勋章」「探索家勋章」「小小旅行家认证爸妈」
- **学马蜂窝蜂首俱乐部私域社群**：「童慧行认证爸妈」群组，给荣誉 + 流量扶持 + 官方活动优先权

### 启示 4：行程规划器接入孩子维度（学携程行程助手 + 穷游 plan）

**携程行程助手**：覆盖行前/行中/行后，值机/登机牌/行李清单/当地紧急号码。
**穷游 plan.qyer.com**：可视化多日行程编辑。
**两家都没考虑孩子**。

→ **童慧行走天下双引擎在行程规划场景要做的 4 件事**：
1. **每日孩子体力曲线**：自动避开连续两天暴走
2. **孩子兴趣匹配**：每日目的地命中孩子档案里的兴趣 +1（恐龙/海洋/历史/动物）
3. **儿童动线**：自动剔除不友好设施（夜场/陡坡/无母婴室）
4. **孩子说反向输入行程**：孩子吐槽「这个景点不好玩」→ 下次行程自动降权

把 `ChildProfile` + `ChildFeelingProfile` 的数据资产在行程规划场景变现。

### 启示 5：孩子说做成「小红书专辑」+「Tripadvisor Travel Lists」混合体

**小红书专辑化收藏**：用户建「0-3 岁带娃景点」「上海周末遛娃」「3 岁生日旅行」专辑——把收藏变成二次创作，是极高留存设计。
**Tripadvisor Travel Lists / Collections**：轻量 UGC 策展——用户发布「我家孩子最爱 10 个景点」。

→ **童慧行走天下的孩子说增强**：
- **专辑化收藏**：妈妈可以把孩子的童言趣语按专辑归档（0-3 岁 / 3-6 岁 / 上海周末 / 长途旅行）
- **一键生成 Travel List**：把孩子的「想去」自动聚合成可分享清单
- **关联勋章**：发布 N 个专辑 / 被收藏 N 次 → 解锁勋章
- 一举占小红书和 Tripadvisor 两个山头
