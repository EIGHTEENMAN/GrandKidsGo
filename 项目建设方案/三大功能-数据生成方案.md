# 三大用户功能：数据生成方案

> 儿童画廊（照片）、足迹地图（打卡）、孩子说（语录）
> 数据来源：**自动提取**（从用户写的攻略/地点的内容中自动识别）+ **用户手动录入**（表单/拍照）
> 最后更新：2026-07-23

## 1. 总览

| 功能 | 自动提取来源 | 手动录入方式 | 当前实现 |
|------|------------|------------|---------|
| **儿童画廊** | Guide.coverImages + contentHtml 中的 `<img>` 标签 | OSS 直传（拍照） + caption 编辑 | POST /api/gallery（手动）✅ |
| **孩子说** | Guide.contentHtml + Spot 字段中引语句子 | 文字输入 + 表情标签 + 录音 | POST /api/child-sayings（手动）✅ |
| **足迹地图** | Guide.cityId → 城市级 + PlanRecord.timelineBlocks 中的 spotId | 地图打卡签到 | 未开发（等 AMAP_API_KEY） |

## 2. 自动提取（核心差异优势）

### 2.1 孩子说：从攻略正文自动提取

用户在发布攻略时，正文（`Guide.contentHtml`）中可能包含孩子的原话。

**规则**：
- 正则匹配带引号的短句（`"……"` / `「……」` / `（孩子说：……）`）
- 句子长度 10-100 字 = 疑似童言趣语
- 带有"说/喊/叫/问/答/告诉妈妈/回头说"等关键词 + 引号 = 高置信度

**流程图**：
```
用户发布攻略 POST /api/guides
        ↓
后端接收 contentHtml（HTML 字符串）
        ↓
自动提取模块 poem-chunks/extract-child-sayings.ts
  ├─ 正则匹配 `[说|喊|叫|问|答|告诉|回头说]："([^"]{10,100})"`
  ├─ 匹配 `「[^」]{10,100}」`
  ├─ 匹配 `（孩子说：[^）]{10,100}）`
  └─ 置信度阈值：至少 2 个中文引号字符
        ↓
发现 → 自动创建 ChildSaying（userId=authUserId, childId=guide.childAges[0] 对应孩 子, source='auto_extract', status='draft')
未发现 → 跳过，无操作
        ↓
用户可以在"孩子说"页面审核自动提取的内容 → 手动确认发布
```

**Schema 改动**（ChildSaying 加字段）：
```prisma
model ChildSaying {
  // 现有字段
  source       String   @default("manual") @map("source")  // manual | auto_extract
  sourceGuideId String? @map("source_guide_id")            // 从哪篇攻略提取的
  status       String   @default("published")              // draft(待审核) | published | hidden
}
```

### 2.2 儿童画廊：从攻略封面 + 正文图片自动收录

**规则**：
- 攻略的 `coverImages[0..2]` 自动收录到儿童画廊（标注 `source='auto_extract'`）
- `contentHtml` 中的 `<img>` 标签提取 src URL（标注 `source='auto_extract'`）
- 自动匹配 `guide.childAges` → `childId`

**流程图**：
```
用户发布攻略
        ↓
后端提取 coverImages 中的图片
        ↓
遍历 contentHtml 中所有 <img src="...">
        ↓
对每张图片 cre ate 一条 PlanMedia（sourceType='gallery', source='auto_extract'）
        ↓
用户可在 /gallery 页面查看 + 编辑 caption + 设为'正式入册'
```

**Schema 无需改动**（PlanMedia 已有 `sourceType='gallery'`、`childId`）

### 2.3 足迹地图：从攻略城市 + 地点列表自动回填

**规则**：
- `Guide.cityId` → 城市级足迹（"去过北京"）
- `PlanRecord.timelineBlocks` 中的 `spotId` → 景点级足迹（"去过北京动物园"）
- 去重：同 `userId + cityId` / `userId + spotId` 不重复创建

**流程图**：
```
用户 fork 攻略 + 完成出行（PlanRecord.status='completed'）
        ↓
从 timelineBlocks 逐块提取 spotId/restaurantId/hotelId
        ↓
自动创建 Footprint（userId, childId, spotId/cityId, lat/lng, checkinAt=plan.endDate）
        ↓
同时记录城市级足迹（cityId）
        ↓
足迹地图自动整理成"旅行地图"可视化
```

**Schema 无需改动**（待 AMAP_API_KEY 到位后新建 `Footprint` 表）

## 6. 风险

- **自动提取质量**：正则可能误匹配（如对话体的攻略正文）。设 `status='draft'`，用户手动确认后才公开
- **图片版权**：自动提取的 `<img>` 如果是外链 URL（如 Unsplash），OSS 自存后再入库
- **childId 匹配**：自动提取时可能无法确定具体哪个孩子，先默认取 `childAges[0]`
