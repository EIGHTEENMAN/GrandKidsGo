童慧行走天下实施方案（v2.0 勋章 + 排行榜 + 社区版）

版本：v2.0
最后更新：2026-07-18
维护者：童慧行

更新摘要：

本版本在 v1.5（2026-07-17 孩子真实感受数据资产版）基础上，把数据资产从"内循环"（画像→拼装→攻略）扩展为"外循环"（勋章→排行榜→社区→兑换→主站联动）。这是走天下从工具向平台的跃迁。

v2.0 钉死的核心命题：

其一，勋章从 13 枚扩到 30+ 枚，覆盖城市探索 / 内容产出 / 社交影响 / 父母成长 / 季节限定 / 亲子协作 六大类。引入稀有度（铜/银/金/钻石），让勋章有收集深度。

其二，排行榜上线。分 4 类（妈妈榜 / 孩子榜 / 城市榜 / 攻略榜）×3 段时间（周/月/总）。孩子真实感受分加权——这是 v1.5 资产在排行榜上的第一次外露。

其三，社区动态。勋章获得 / 攻略发布 / 出行完成三件事变成 feed 流，用户可互相关注、点赞、评论。这是 v1.5 私密圈层的破圈。

其四，兑换体系。勋章可换主站积分（与 store 打通），形成"内容→资产→积分"的闭环。

v2.0 钉死的硬约束（v1.4 第十五节 B 不变）：

其一，排行榜不显示孩子真实姓名。统一用"宝宝 {N} 月"格式。
其二，勋章可展示，但具体行程内容（孩子照片 / 坐标 / 真实时间）在公开场景全部脱敏。
其三，未成年人数据主权不变：监护人同意 3 年留痕，撤回链路级联清除（沿用 v1.4 附录 G）。
其四，孩子真实感受数据**不**参与排行公开（仅"评分均值"可参与），原始 ChildRating 不可见。
其五，UGC 范围保持严格：勋章/排行榜/社区都只展示用户已授权的内容。撤回即刻消失。

主要改动：

第一节补充"V2.0 从工具到平台"段落。
第二节加入"孩子保护优先级跃升"：孩子数据是 v1.5 内循环，v2.0 开始外循环，隐私护栏强度同步升级。
第三节新增 6 张表：TravelLeaderboardSnapshot / TravelActivity / TravelFollowRelation / TravelBadgeExchange / TravelBadgeViewLog / TravelPrivacySetting。
第四节新增 mobile 页面：leaderboard / community / activity / profile（用户主页）。
第五节引擎 B 加第 7 类"勋章触发预测"——基于孩子画像预测未来 30 天可解锁的勋章，主动推送。
第八节勋章体系全部重写。
新增"第九节 B 排行榜系统"。
新增"第九节 C 社区动态 + 关注系统"。
新增"第九节 D 兑换体系"。
新增"第十三节 B 跨产品主站联动 v2"。

v1.0 ~ v1.5 五份文档保留作历史参考，本文档为后续开发与验收的唯一文本。

本方案为内部项目实施文档，面向产品研发交付团队。文档遵循项目开发规则 4.3 节纯文本规范书写，不含 Markdown 格式符号。

第一节 战略意义与产品主链路
================================================================

一、v1.5 复盘

v1.5 走天下定位为"妈妈视角的亲子出行决策工具"——拼装引擎 A + 引擎 B 智能问答 + 攻略发布 + 数据资产闭环。三城 15 spot 入库，端到端可演示。

v1.5 的数据资产停留在"内循环"：ChildProfile → ChildExpectation → ChildRating → ChildFeelingProfile → 反哺引擎。用户在产品内看到的是"我孩子的感受画像"，但这些数据**没有外露**——别人看不到你孩子去过哪儿、玩得开不开心。

二、v2.0 跃迁：从工具到平台

v2.0 引入"外循环"：用户的勋章 / 攻略 / 出行成果**可选择**对外展示（默认严格脱敏），形成可比较、可关注、可兑换的社交层。

外循环的护城河：
其一，孩子真实感受分 = 排行榜的**核心权重**。这是其他平台没有的独家信号——小红书的"评分"是主观分，孩子真实感受是结构化 7 维度。
其二，"宝宝 {N} 月" + 城市 + 主题 + 真实感受分（均值），构成去隐私的对比维度——妈妈可以看到"和我孩子差不多月龄的小朋友，在杭州玩了 4 个项目，平均开心度 4.6/5"。
其三，社交激励放大"内容产出"动机——妈妈写攻略的动力，从"自己留念"升级为"获得勋章 + 上榜 + 兑换积分"。

v2.0 的三大产品价值承诺（与 v1.5 六条并列）：

承诺七，勋章是真心荣誉，不是装饰。每一枚勋章都对应一个**结构化行为**（v1.5 钉死的"勋章检测引擎"），无法刷、不能买、不靠运营刷量。
承诺八，排行榜体现真实用户价值，不靠流量。所有排行都以"孩子真实感受分"为基底权重，刷流量不刷分。
承诺九，孩子数据是孩子自己的，平台只做连接者。家长是孩子的代理人，但 8 岁以上孩子有发言权（v1.4 第十五节 B 第七点不变），社区任何展示都可由家长一键撤回。

不做：广告位、不做商家推广、不做 PGC 内容代发。

不做：跨 app 排行榜（不走天下独立排名，不与主站学习榜 / 挑战榜打通）。

第二节 用户定位与价值主张（v2.0 隐私跃迁）
================================================================

一、用户优先级（v1.5 不变）

孩子体验 > 母亲体验 > 父亲体验

v2.0 加入第四条硬规则：

孩子数据主权 > 平台内容运营 > 任何商业化诉求

任何"为了产品增长"想突破这条规则的需求，必须在方案文档里单列"对孩子的影响"段落 + 法务 + 童慧行品牌方三重签字。

二、隐私跃迁：从内循环到外循环

v1.5 孩子真实感受数据是"内循环"——只产品内使用。

v2.0 这些数据**开始有外露**：
其一，勋章展示（只显示勋章名 + 描述，不显示具体行程）。
其二，排行榜（显示"宝宝 {N} 月" + 城市 + 主题 + 真实感受分均值，不显示单条评分）。
其三，社区动态（只显示"用户 X 在杭州获得 3 枚勋章"，不显示具体内容）。
其四，主站联动（个人中心的"走天下记录"沿用 v1.5 脱敏规则）。

外露必须满足三件事：
其一，行为前必须主动同意（沿用 v1.4 第十五节 B 第二点：3 年留痕，scope = `publish_to_public`）。
其二，撤回可级联清除（沿用 v1.4 第六点）。
其三，未成年人专属撤回：8 岁以上孩子可以主动通过客服拒绝（24 小时内必须处理）。

三、用户分层（v2.0 新增"成长型妈妈"画像）

v1.5 用户分 3 类：浏览型 / 创建型 / 内容型。

v2.0 加第 4 类："成长型妈妈"——以孩子的真实感受变化为驱动力。她们不太写攻略，但会持续记录孩子的评分（即使是简化的 5 星），并希望看到"和我孩子差不多月龄的孩子，玩同一个地方会开心吗"。

这一类用户是 v2.0 排行榜 + 社区的主要受众：
她们不生产攻略，但是排行榜的"投票者"（通过浏览、点赞、收藏间接投票）。
她们不写勋章墙分享，但是勋章动态的"看客"——看到关注的人解锁新勋章，自己也想解锁。

四、价值主张（v2.0 增量）

对"创建型"用户（v1.5 主目标）："你的攻略 + 真实记录 = 排名 + 勋章 + 积分。"
对"成长型"用户（v2.0 新目标）："看看和你孩子差不多大的小朋友，在哪儿玩得开心。"
对"内容型"用户（v1.5 主目标）："勋章是真心荣誉，每一枚都对应真实行为。"

第三节 数据模型
================================================================

一、v1.5 已有表（保留 + 必要扩展）

v1.5 共 33 张表，v2.0 全部保留。新增字段：

UserFollow（v1.5 已有）扩展：增加 `muted: bool` 字段——用户可静默关注某人不被对方发现。

TravelBadgeDef（v1.5 已有）扩展：
新增 `rarity: enum(bronze|silver|gold|diamond)`。
新增 `tier: int`（0-4，用于排序）。
新增 `seasonalTag: string?`（如"暑期限定"）。
新增 `hiddenFlag: bool`（隐藏勋章，达到条件才出现）。

TravelBadge（v1.5 已有）扩展：
新增 `shareScope: enum(private|public|community)`——每枚勋章可独立设置公开范围。
新增 `sharedAt: datetime?`——首次公开分享时间。

Guide（v1.5 已有）扩展：
新增 `likeCount` 已存在。增加 `viewCount30d` 缓存字段（排行榜用，避免实时 count）。

二、v2.0 新增 6 张表

TravelLeaderboardSnapshot（排行榜快照）
- 周期：每日 02:00 跑。
- 字段：scope (mom|child|city|guide)、period (week|month|all)、payloadJson (排行榜条目 JSON)、capturedAt、weekKey (如"2026-W28")。
- 关键索引：(scope, period, capturedAt DESC)。

TravelActivity（社区动态）
- 字段：id、userId、type (badge_unlocked|guide_published|trip_completed)、targetId、createdAt、contentJson (动态文案模板渲染后的内容)、isPublic。
- 关键索引：(isPublic, createdAt DESC)；(userId, createdAt DESC)。

TravelFollowRelation（v1.5 UserFollow 升级，存为新表避免破坏原 schema）
- 沿用 v1.5 的 followerId + followeeId，但增加：muted bool、createdAt。
- v2.0 不合并 v1.5 UserFollow：v1.5 那个用于"是否在关注"判断，v2.0 这个用于"社区可见 + 静默"。

TravelBadgeExchange（兑换记录）
- 字段：id、userId、badgeId、exchangedAt、pointsAwarded、status (pending|completed|reverted)。
- 关键索引：(userId, exchangedAt DESC)。

TravelBadgeViewLog（勋章曝光日志）
- 用于"勋章触发预测"（v2.0 引擎 B 第 7 类）——记录用户看过的勋章墙 / 动态内容，反哺推荐。
- 字段：id、userId、viewedBadgeDefId、viewedAt、context (page|feed|search)。

TravelPrivacySetting（用户隐私偏好）
- 字段：userId、allowLeaderboardPublic (bool)、allowCommunityFeed (bool)、badgeShareScope (默认 private)、updatedAt。
- 关键索引：userId unique。

三、v1.5 隐私合规（v2.0 不变 + 加强）

v1.4 第十五节 B 十条硬合规 + 附录 G 红线手册全部保留。v2.0 在以下场景加强：
其一，勋章分享前必须显式同意——v1.5 是"按 plan 同意"，v2.0 是"每枚勋章独立同意"（更细粒度）。
其二，排行榜加入"按周匿名" 选项——用户可选择"不参与公开榜"，但仍可看自己的排名。
其三，导出数据包（GDPR-like）保留——用户可一键导出"我孩子在走天下的所有数据 JSON"。

第五节 拼装引擎与 AI 能力边界
================================================================

一、引擎 A 拼装（v1.5 不变）

三档候选 / 100ms 内 / 6 要素加权。

v2.0 引擎 A 增量：拼装完成后，额外返回"预测勋章"——告诉用户"如果你完成这次出行，将解锁 N 枚勋章"（v2.0 引擎 B 第 7 类）。

二、引擎 B 五类能力（v1.5 不变）

行程智能问答、行程微调、AI 润色、人脸检测/声纹抽取（v1.5 第 4 类）、感受匹配问答。

v2.0 加第 6 类：勋章触发预测。

输入：用户画像 + 已完成计划 + 已获得勋章。
输出：未来 30 天可解锁的勋章列表 + 概率 + "如何达成"建议。

第九节 B 排行榜系统（v2.0 新增）
================================================================

一、4 类排行榜

1. 妈妈榜（Mama Leaderboard）
   - 排名依据：综合得分（详见附录 B 公式）
   - 展示字段：排名 / 昵称（脱敏）/ 城市 / 勋章数 / 真实感受分均值 / 子女数
   - 隐私：子女数 → "亲子 N" 形式，不显示月龄

2. 孩子榜（Kid Leaderboard）
   - 排名依据：孩子的真实感受分均值
   - 展示字段：排名 / "宝宝 {N} 月" / 城市 / 主题 / 真实感受分均值 / 探索城市数
   - 隐私：孩子昵称 = "宝宝" + 月龄

3. 城市榜（City Leaderboard）
   - 排名依据：城市被探索的次数 + 平均真实感受分
   - 展示字段：排名 / 城市名 / 探索次数 / 平均真实感受分 / 推荐主题

4. 攻略榜（Guide Leaderboard）
   - 排名依据：攻略的"真实感受分贡献"加权（v1.5 数据资产）
   - 展示字段：排名 / 攻略标题 / 作者（脱敏）/ 真实感受分 / 收藏数 / 点赞数

二、3 个时间维度

本周（week）：每周一 00:00 切榜
本月（month）：每月 1 号 00:00 切榜
总榜（all）：所有时间累积

三、评分公式（详见附录 B）

基础公式：

  score = (0.5 * feelingScore + 0.2 * badgeCount + 0.15 * guideCount + 0.15 * cityCount) * decayFactor

  其中：
  - feelingScore = 孩子真实感受分均值（0-5）
  - badgeCount = 已解锁勋章数（按稀有度加权：铜=1 / 银=2 / 金=3 / 钻石=5）
  - guideCount = 已发布攻略数
  - cityCount = 探索城市数
  - decayFactor = exp(-0.05 * daysSinceLastAction) （半衰期 ~14 天）

反作弊规则：
其一，10 天内单日评分 > 50 条 → 该用户该周分数减半。
其二，单条攻略 7 天内获 > 100 收藏 + 0 评论 → 标记异常，下周不参与。
其三，连续 3 周排名 Top 10 但感受分均值 < 3 → 警告 + 不参与公开榜。

四、排行榜快照（性能设计）

不实时计算。每日 02:00 跑一次，存 TravelLeaderboardSnapshot。
API 直接读快照，毫秒级返回。
榜单刷新有抖动是允许的（每天 02:00 后展示新榜）。

五、隐私护栏

其一，未授权用户（没勾"参与公开榜"）不进入榜，但可查看自己排名。
其二，孩子月龄显示为"宝宝 {N} 月"，最小粒度 3 个月（如 3 月 / 6 月 / 9 月 / 12 月 / ...）。
其三，公开榜不显示用户头像，仅显示昵称首字。
其四，撤回授权后，下次刷新即从榜中移除。

附录 A 数据 schema 详细字段说明
================================================================

一、TravelLeaderboardSnapshot

字段：id (uuid)、scope (mom|child|city|guide)、period (week|month|all)、
payloadJson (text, 排行榜 JSON)、capturedAt (datetime)、weekKey (text, ISO week like "2026-W28")。

索引：(scope, period, capturedAt DESC)。

二、TravelActivity

字段：id (uuid)、userId (uuid)、type (badge_unlocked|guide_published|trip_completed)、
targetId (text)、contentJson (text, 渲染后的文案)、isPublic (bool)、createdAt (datetime)。

索引：(isPublic, createdAt DESC)；(userId, createdAt DESC)。

三、TravelFollowRelation

字段：id (uuid)、followerId (uuid)、followeeId (uuid)、muted (bool)、createdAt (datetime)。

索引：(followerId, followeeId) unique；(followeeId, createdAt DESC)。

四、TravelBadgeExchange

字段：id (uuid)、userId (uuid)、badgeDefName (text)、pointsAwarded (int)、
status (pending|completed|reverted)、exchangedAt (datetime)。

索引：(userId, exchangedAt DESC)；(status, exchangedAt)。

五、TravelBadgeViewLog

字段：id (uuid)、userId (uuid)、viewedBadgeDefName (text)、viewedAt (datetime)、
context (page|feed|search)。

索引：(userId, viewedAt DESC)；(viewedBadgeDefName, viewedAt)。

六、TravelPrivacySetting

字段：userId (uuid unique)、allowLeaderboardPublic (bool)、allowCommunityFeed (bool)、
badgeShareScope (private|public|community)、updatedAt (datetime)。

附录 B 排行榜评分公式
================================================================

一、妈妈榜（Mama Leaderboard）

score_mama = (0.5 * feeling_score_avg + 0.2 * weighted_badge_count + 0.15 * guide_count + 0.15 * city_count) * decay

  其中：
  - feeling_score_avg = 孩子真实感受分均值（所有计划加权平均，0-5）
  - weighted_badge_count = sum(rarity_weight * count)
      rarity_weight: bronze=1, silver=2, gold=3, diamond=5
  - guide_count = 已发布攻略数
  - city_count = 探索城市数（去重）
  - decay = exp(-0.05 * days_since_last_action) // 半衰期 ~14 天

二、孩子榜（Kid Leaderboard）

score_kid = feeling_score_avg * (1 + 0.1 * city_count) * decay

  - 孩子榜只看真实感受分，权重 = 平均分 × 城市多样性
  - 反作弊：单日评分 > 20 条，该日分数按 0.5 计算

三、城市榜（City Leaderboard）

score_city = log(trip_count + 1) * (0.7 + 0.3 * feeling_score_avg)

  - 城市榜按被探索次数 × 平均分
  - 对数增长防止头部城市垄断

四、攻略榜（Guide Leaderboard）

score_guide = 0.4 * feeling_score_contribution + 0.3 * save_count + 0.2 * like_count + 0.1 * view_count_recent_30d

  - feeling_score_contribution = 该攻略触发的孩子真实感受分均值

五、反作弊规则

R1：单日评分 > 50 条 → 该周分数 × 0.5
R2：单条攻略 7 天内 > 100 收藏 + 0 评论 → 标记异常，下周不参与
R3：连续 3 周 Top 10 但 feeling_score_avg < 3 → 警告 + 不参与公开榜
