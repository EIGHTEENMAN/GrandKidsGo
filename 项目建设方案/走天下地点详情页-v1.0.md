# 走天下地点详情页 v1.0 全栈改造方案

> 项目：apps/travel-guide（Next.js 14 + Prisma + PostgreSQL），部署域名 `travel.grandand.com`
> 路由：`/place/[type]/[id]`，API：`/api/places/[type]/[id]`
> 目标：橙色 emoji 老 UI → 蓝青渐变 + 全 SVG + 大图轮播 + 周边 13-15 类 + 交通 4 块 + 榜单 + 攻略关联
> 已锁定的关键约束：UI 调性（`from-blue-600 via-cyan-600 to-teal-600`）、零 emoji、Unsplash 兜底图、客观字段（地址/电话/官网/营业时间）走结构化字段而非 AI 起草

## Context

当前详情页 (`apps/travel-guide/src/app/place/[type]/[id]/page.tsx`) 是 v3.0 橙色 emoji 老 UI（`bg-gradient-to-r from-amber-500 to-orange-500` + 👩👶🅿️🪑🍼🚼 全套 emoji），缺少图片、周边 POI、交通信息、榜单位置、攻略关联。用户要求按"孩子需求"角度重做：UI 全面对齐当前 v4.0 蓝青渐变（`from-blue-600 via-cyan-600 to-teal-600`）、大图轮播、零 emoji，并展示**13-15 类孩子需求周边 POI** + **4 块交通信息** + **榜单位置**。

数据库现状：
- 7 类地点表都有 `lat/lng`，但除 `Hospital` 外**全部缺 address/phone/openHours**
- `Spot` 有 `images String[]`，但 `04-import-db.ts` 永远写 `[]`；其他 6 类无图字段
- **无 `PlaceNearby` / `PlaceTransport` 表**
- `TravelLeaderboardSnapshot` 不含 place；place-hot 是实时查 PlaceReview top 8

实施范围：**全栈改造**（schema + API + UI）。分 P0-P4 渐进上线，P0 先发保证体感完整（mock 数据 + Unsplash 兜底），后续 P1-P4 替换为真实数据。

## 1. 数据库 Schema 改动（`prisma/schema.prisma`）

### 1.1 7 类地点表统一加基础字段

```prisma
  BOOKSTORE            // 儿童书店/绘本馆
  KIDS_PHOTO           // 儿童摄影/照相馆
  KIDS_HOSPITAL        // 儿童医院/儿科诊所
  PHARMACY             // 药店
  MATERNITY_STORE      // 母婴店
  DIDI_PICKUP          // 网约车上下车点
  TAXI_STAND           // 出租车候车区
  BIKE_SHARE           // 共享单车/共享婴儿车
  KID_HOTEL            // 亲子酒店（家庭房/儿童泳池/儿童早餐）—— 2026-07-22 用户评估补充
  AGE_SUITABLE         // 适龄推荐（spot.recommendedMonths 关联）
  STROLLER_FRIENDLY    // 婴儿车可达路径标记（已存在，独立保留）
  RAIN_FRIENDLY        // 雨天友好 tag 标记（不独立板块，仅做标签）—— 2026-07-22 用户评估决策

model PlaceNearby {
  id             String              @id @default(uuid())
  placeId        String              @map("place_id")
  placeType      String              @map("place_type")        // 'sight' | 'restaurant' | ...
  category       PlaceNearbyCategory
  name           String
  lat            Float?
  lng            Float?
  distanceMeters Int?                @map("distance_meters")
  extra          Json                @default("{}")            // 品类差异化字段
  source         String              @default("ai_draft_v1")   // ai_draft_v1 | official | user_contrib
  isVerified     Boolean             @default(false) @map("is_verified")
  createdAt      DateTime            @default(now()) @map("created_at")

  @@index([placeId, placeType, category])
  @@index([category])
  @@map("place_nearby")
}
```

`extra` JSON 示例：
- `KID_RESTAURANT`: `{ "hasKidsMenu": true, "avgPrice": 80, "walkInOk": true }`
- `NURSING_ROOM`: `{ "isFree": true, "hasHotWater": true }`
- `DIDI_PICKUP`: `{ "notes": "南门出来 30 米路北" }`

### 1.3 新增 `PlaceTransport` 交通表

```prisma
enum TransportKind {
  SUBWAY             // 地铁站
  BUS                // 公交站
  PARKING            // 停车场
  AIRPORT            // 机场
  TRAIN_STATION      // 火车站/高铁站

model PlaceTransport {
  id             String         @id @default(uuid())
  placeId        String         @map("place_id")

  kind           TransportKind
  name           String                                 // "地铁 1 号线 天安门东站 B 口"
  lineOrLabel    String?        @map("line_or_label")  // "1 号线" / "首都机场线"

  details        Json           @default("{}")          // { "exitNotes": "B 口出有电梯", "parkingSpaces": 120 }

  createdAt      DateTime       @default(now()) @map("created_at")

  @@index([placeId, placeType, kind])
  @@index([kind])
  @@map("place_transports")
```


### B. 大图轮播图片（heroImages）

**当前状态**：`Spot.images String[]` 字段永远写 `[]`，前端 `coverImage: s.images?.[0] ?? null` 全是 null。

**方案**：短期用 Unsplash CDN 预定义图片兜底（不依赖任何服务端调用）。

注意：Unsplash Source API（`source.unsplash.com`）自 2024-06 已停止维护。**改用直接 Unsplash CDN**（固定 photo ID），预定义 5 张亲子旅行主题图，按 `place.id` hash 选不同子集：

```ts
// apps/travel-guide/src/lib/place-hero.ts（新增工具）
const HERO_POOL = [
  'photo-1602002418082-a4443e081dd1',  // 亲子海边
  'photo-1511895426328-dc8714191300',  // 真实家庭
  'photo-1502086223501-7ea6ecd79368',  // 孩子手
  'photo-1559131397-f94da358f7ca',     // 阳光下亲子
  'photo-1545569310-c55b3c63b8c2',     // 公园玩耍
];

export function buildHeroImages(place: { id: string; coverImages: string[] }): string[] {
  // 1. 数据库有图就用 db
  if (place.coverImages?.length > 0) {
    return place.coverImages.slice(0, 3);
  }
  // 2. 按 placeId hash 从 POOL 选 3 张
  const hash = Array.from(place.id).reduce((s, c) => s + c.charCodeAt(0), 0);
  return [0, 1, 2].map(i => `https://images.unsplash.com/${HERO_POOL[(hash + i) % HERO_POOL.length]}?w=1600&q=85`);
}
```

**实施时机**：P0 立即（前端函数，0 后端成本）。P4 中期：写 `03-fetch-photos.ts` 调高德 `v3/place/detail` 拿 photos 字段入 `coverImages`。

```ts
// apps/travel-guide/src/lib/data-pipeline/02b-amap-around.ts
import { createAmapClient } from './_amap-client';
import { PrismaClient } from '@prisma/client';

const NEARBY_TYPE_CODES: Record<PlaceNearbyCategory, { keywords: string; typecodes?: string[] }> = {
  KID_RESTAURANT: { keywords: '亲子餐厅', typecodes: ['050100', '050101'] },     // 高德 typecode
  NURSING_ROOM:   { keywords: '母婴室' },
  TAP_WATER:      { keywords: '直饮水' },
  CONVENIENCE:    { keywords: '便利店', typecodes: ['060100'] },
  TOY_STORE:      { keywords: '玩具反斗城|乐高|泡泡玛特' },
  BOOKSTORE:      { keywords: '儿童书店|绘本馆' },
  KIDS_PHOTO:     { keywords: '儿童摄影|照相馆' },
  KIDS_HOSPITAL:  { keywords: '儿童医院|儿科', typecodes: ['090100'] },
  PHARMACY:       { keywords: '药店', typecodes: ['090200'] },
  MATERNITY_STORE:{ keywords: '母婴店' },
  DIDI_PICKUP:    { keywords: '网约车' },
  TAXI_STAND:     { keywords: '出租车候车区' },
  BIKE_SHARE:     { keywords: '共享单车' },
};

// 用 haversine 算距离（复用 assembler/index.ts:456-470 的函数）
function haversineMeters(a: {lat: number; lng: number}, b: {lat: number; lng: number}): number { ... }

async function fillNearbyForCity(city: CityMeta) {
  const amap = createAmapClient();
  const spots = await prisma.spot.findMany({ where: { cityId: city.id } });
  for (const spot of spots) {
    if (!spot.lat || !spot.lng) continue;
    for (const [category, cfg] of Object.entries(NEARBY_TYPE_CODES)) {
      const pois = await amap.around({
        keywords: cfg.keywords,
        location: `${spot.lng},${spot.lat}`,
        radius: 2000,                              // 2km
        types: cfg.typecodes?.join(','),
        offset: 5,
      });
      for (const p of pois) {
        const [lng, lat] = p.location.split(',').map(Number);
        await prisma.placeNearby.upsert({
          where: { id: `${spot.id}-${category}-${p.id}` },
          create: {
            placeId: spot.id, placeType: 'sight',
            category, name: p.name, lat, lng,
            distanceMeters: Math.round(haversineMeters(
              { lat: spot.lat!, lng: spot.lng! },
              { lat, lng }
            )),
            extra: {},
            source: 'official',
            isVerified: true,
          },
          update: {},
        });
      }
    }
  }
}
```

**配额评估**：高德 `around` 端点 5000 次/日（`_amap-client.ts:59`）。每城每 spot × 13 类 ≈ 13 calls；50 城 × 10 spot/城 = 6,500 calls/全量回填。**分批跑**：每次跑 5 城，1 周内覆盖 50 城。

**实施时机**：P1 短期 mock seed；P4 中期实数据回填。

### E. 榜单位置（leaderboard.rank）

**当前状态**：`/api/leaderboard/place-hot?period=week` 返回 top 8；无单地点 rank 查询。

**方案**：新增 `/api/places/[type]/[id]/leaderboard/route.ts`，**服务端转发**该 placeId 在 place-hot 列表中的 index：

```ts
// apps/travel-guide/src/app/api/places/[type]/[id]/leaderboard/route.ts
import { NextResponse } from "next/server";

export async function GET(req: Request, { params }: { params: { type: string; id: string } }) {
  const base = new URL(req.url).origin;
  const r = await fetch(`${base}/api/leaderboard/place-hot?period=week`, { cache: 'no-store' });
  const d = await r.json();
  const items = d.data?.items ?? [];
  const idx = items.findIndex((i: any) => i.placeId === params.id);
  return NextResponse.json({
    code: 'OK',
    data: {
      rank: idx === -1 ? null : idx + 1,
      total: items.length,
      scope: 'place-hot',
      period: 'week',
    },
  });
}
```

**实施时机**：P3 立即可用（纯转发，不动数据）。

```ts
    dataWarnings: { nearbyIsMock: boolean, transportIsMock: boolean },
  }
}
```

实现核心：`Promise.all` 并发 5 个子查询（reviews/nearby/transport/leaderboard fetch/relatedGuides），p95 < 300ms。**不拆端点**，避免首屏拖慢。

## 7. 风险 & 取舍

- **mock 数据诚实问题**：DB `source='ai_draft_v1' || 'mock'` + `isVerified=false`，后端 `dataWarnings` 透传前端，触发顶部 banner"以下周边/交通为示意数据，以现场为准"。P4 真数据回填后自动消失。
- **图片版权**：P0 用 Unsplash CDN + 5 张预定义亲子旅行图（Unsplash License 允许商用）；P4 从高德 `v3/place/detail` photos 字段 + OSS 自存。
- **性能**：单端点 30KB JSON（reviews 30 条 ~10KB + nearby ~5KB + transport ~3KB + 其他 ~12KB），gzip 后 ~6KB；`Promise.all` 5 子查询 p95 < 300ms；详情页加 `revalidate: 60` SWR 缓存。
- **不用单表继承**：Prisma 单表继承会让所有地点变成 `Place` 大表，丢失 Spot.momHighlights/kidHighlights 这种语义字段，得不偿失。
- **PlaceNearby 用单表 + enum**：避免 13 张表 join 噩梦；后期新增"直饮水"等品类只动 enum 不动 schema。
